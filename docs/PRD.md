# Image2Client 产品需求文档（PRD）

> **一个专门为 GPT Image / OpenAI Image API 打造的专业客户端。**
>
> 不是聊天软件，也不是 AI 聚合平台，而是一个类似 **Postman + 图片工作台** 的工具。

---

## 1. 产品概述

### 1.1 产品定位

面向个人开发者/设计师的本地化 OpenAI Image API 客户端，支持所有兼容 OpenAI Image API 的中转站。全部数据保存在本地，无需后端服务。

### 1.2 目标用户

- 使用 OpenAI Image API 或兼容中转站的开发者
- 需要批量生成/编辑商品图、海报等素材的设计师
- 对数据隐私有要求、不希望数据上传第三方服务器的用户

### 1.3 核心价值

- 零后端：纯前端 SPA，打开即用
- 数据本地化：API Key、图片、历史记录全部保存在浏览器本地
- 兼容性强：支持任何兼容 OpenAI Image API 的 Base URL

---

## 2. 技术架构

### 2.1 技术栈

| 层级 | 选型 | 说明 |
|------|------|------|
| 框架 | React + Vite | 轻量、开发快 |
| 样式 | Tailwind CSS | 实用优先的原子化 CSS |
| 组件库 | shadcn/ui | 现代、可定制 |
| 本地存储（配置） | LocalStorage | 保存 BaseURL、API Key、偏好配置 |
| 本地存储（数据） | IndexedDB | 保存历史图片、Prompt、生成记录 |

### 2.2 不需要

- ❌ Node.js 后端
- ❌ 数据库服务
- ❌ 用户系统 / 登录注册
- ❌ 云存储 / OSS

### 2.3 数据流

```
用户配置（BaseURL / APIKey / 偏好）  →  LocalStorage
图片 / Prompt / 历史记录             →  IndexedDB
API 请求                             →  直接从浏览器发往目标 Base URL
```

## 3. 功能需求

### 3.1 API 配置（P0）

| 字段 | 说明 |
|------|------|
| Base URL | 支持自定义中转站地址 |
| API Key | 加密存储于 LocalStorage |
| Model | 下拉选择，默认 `gpt-image-2` |
| 一键测试连接 | 调用轻量接口验证配置是否可用 |

**验收标准：**
- 配置保存后刷新页面不丢失
- 测试连接成功/失败有明确提示
- API Key 输入框支持显示/隐藏切换

---

### 3.2 图片生成 — Text to Image（P0）

| 参数 | 类型 | 说明 |
|------|------|------|
| Prompt | textarea | 支持多行，字数统计 |
| Size | select | `1024x1024` / `1536x1024` / `1024x1536` / 自定义 |
| Quality | select | `auto` / `high` / `medium` / `low` |
| Background | select | `auto` / `transparent` / `opaque` |
| Output Format | select | `png` / `webp` / `jpeg` |
| N | number | 生成数量（1-4） |

**验收标准：**
- 点击 Generate 后显示 loading 状态
- 生成成功后在右侧预览区展示结果
- 支持单张下载、批量下载
- 生成记录自动保存到历史

---

### 3.3 图片编辑 — Image Edit（P0）

| 参数 | 类型 | 说明 |
|------|------|------|
| 原图 | file upload | 支持拖拽上传，预览 |
| Mask | file upload（可选） | 遮罩图，用于指定编辑区域 |
| 参考图 | file upload（多张） | 风格/内容参考 |
| Prompt | textarea | 编辑指令 |
| Size / Quality / Format | 同生成参数 | — |

**验收标准：**
- 上传图片后即时预览
- Mask 支持可选上传
- 参考图支持多张上传（最多 5 张）
- 编辑结果与原图可对比查看

---

### 3.4 历史记录（P0）

**保存内容：**
- Prompt
- 完整请求参数
- 原图（如有）
- 结果图（Base64 或 Blob）
- 生成时间

**支持操作：**
- 一键重新生成（复用参数）
- 删除单条 / 批量删除
- 导出记录（JSON / ZIP）
- 按时间倒序排列
- 搜索（按 Prompt 关键词）

**验收标准：**
- 历史记录持久化存储于 IndexedDB
- 清除浏览器缓存前数据不丢失
- 大量记录时列表加载流畅（虚拟滚动）

---

### 3.5 Prompt 模板（P0）

**功能：**
- 收藏常用 Prompt
- 分类管理（自定义标签）
- 关键词搜索
- 一键应用到输入框

**预置分类示例：**
- 服装
- 商品图
- 海报
- 透明背景
- 人像
- Logo

**验收标准：**
- 支持新增/编辑/删除模板
- 支持自定义分类标签
- 点击模板即填入 Prompt 输入框

---

### 3.6 批量任务（P1 — 后期迭代）

**流程：**
```
上传商品图列表 / 输入多条 Prompt
        ↓
    配置统一参数
        ↓
    循环调用 API（含并发控制）
        ↓
    实时进度展示
        ↓
    自动保存结果 + 支持批量下载
```

**验收标准（后期）：**
- 支持设置并发数
- 单条失败不影响整体任务
- 失败任务支持重试
- 进度条实时更新

## 4. UI 设计

### 4.1 整体布局

```
┌──────────── 左侧面板 ────────────┬─────────── 右侧面板 ───────────┐
│                                  │                                │
│  Prompt 输入区                   │                                │
│  参数配置区                      │        图片预览 / 结果展示      │
│  图片上传区                      │                                │
│  [Generate] 按钮                 │                                │
│                                  │                                │
├──────────────────────────────────┼────────────────────────────────┤
│                                  │                                │
│  历史记录列表                    │    参数详情 / 元信息            │
│  Prompt 模板                     │    下载 / 操作按钮              │
│                                  │                                │
└──────────────────────────────────┴────────────────────────────────┘
```

### 4.2 设计原则

- 深色/浅色主题切换
- 响应式布局（桌面优先）
- 拖拽上传交互
- 图片预览支持缩放、对比
- 操作反馈清晰（Toast / Loading / Skeleton）

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 隐私安全 | 所有数据仅存浏览器本地，不上传任何第三方服务器 |
| 性能 | 历史记录 1000+ 条时仍流畅 |
| 兼容性 | Chrome / Edge / Firefox 最新版 |
| 部署 | 纯静态文件，可部署至 Vercel / Netlify / GitHub Pages |
| 离线能力 | 除 API 调用外，其余功能可离线使用 |

## 7. 项目结构（建议）

```
src/
├── components/       # UI 组件
│   ├── ui/           # shadcn/ui 基础组件
│   ├── layout/       # 布局组件
│   ├── generate/     # 图片生成相关
│   ├── edit/         # 图片编辑相关
│   ├── history/      # 历史记录
│   └── templates/    # Prompt 模板
├── hooks/            # 自定义 Hooks
├── lib/              # 工具函数
│   ├── api.ts        # OpenAI Image API 封装
│   ├── storage.ts    # LocalStorage 封装
│   └── db.ts         # IndexedDB 封装（Dexie.js）
├── stores/           # 状态管理（Zustand）
├── types/            # TypeScript 类型定义
└── App.tsx
```
